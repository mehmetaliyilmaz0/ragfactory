## Goal
Create all 42 Jinja2 templates that `generator.py` renders. After this phase,
`generate()` produces real, runnable Python code for every framework × component
combination. Templates are the public surface of the project — they must be
correct, consistent, and production-grade.

## [FIX C2] Architectural Decision: Stage → Entrypoint Structural Contract

**Decision: Each stage template produces module-level imports + a `def build_<stage>() -> Any:` factory function.**

Rationale:
- Module-level imports are idiomatic Python and readable to the end-user
- Factory functions enable lazy initialization (no API calls at import time)
- `def build_<stage>()` gives test authors a stable symbol to assert on
- Avoids module-level side effects (no credentials read at import)

### Canonical Stage Fragment Shape

Every stage template (all 33 of them) MUST follow this exact shape:

```python
# ── <Stage Title> ─────────────────────────────────────────────────────────────
{% if framework == "langchain" %}
from <langchain_package> import <ClassName>
{% elif framework == "llamaindex" %}
from <llama_index_package> import <ClassName>
{% endif %}


def build_<stage>() -> Any:
{% if framework == "langchain" %}
    return <ClassName>(
        <param>={{ config_value }},
    )
{% elif framework == "llamaindex" %}
    return <ClassName>(
        <param>={{ config_value }},
    )
{% endif %}
```

Rules:
1. One blank line between the import block and the function definition
2. The function name is **exactly** `build_chunking`, `build_embedding`,
   `build_vectordb`, `build_retrieval`, `build_reranker`, `build_llm`
3. Return type annotation is `Any` (imported once at entrypoint level)
4. Config values are always interpolated, never hardcoded defaults

### Canonical Pipeline Entrypoint Shape

`entrypoints/<framework>/pipeline.py.j2` wires all stages together:

```python
"""{{ pipeline_name }} — Generated by ragfactory {{ config.version }}."""
from __future__ import annotations

import os
from typing import Any

# ─── Stage: Chunking ──────────────────────────────────────────────────────────
{{ stages.chunking }}

# ─── Stage: Embedding ─────────────────────────────────────────────────────────
{{ stages.embedding }}

# ─── Stage: Vector DB ─────────────────────────────────────────────────────────
{{ stages.vectordb }}

# ─── Stage: Retrieval ─────────────────────────────────────────────────────────
{{ stages.retrieval }}

{% if stages.reranker is defined and stages.reranker %}
# ─── Stage: Reranker ──────────────────────────────────────────────────────────
{{ stages.reranker }}

{% endif %}
# ─── Stage: LLM ───────────────────────────────────────────────────────────────
{{ stages.llm }}


def run_pipeline(query: str) -> str:
    """Run the full {{ pipeline_name }} RAG pipeline."""
    embedder    = build_embedding()
    vectorstore = build_vectordb(embedder)
    retriever   = build_retrieval(vectorstore)
{% if stages.reranker is defined and stages.reranker %}
    retriever   = build_reranker(retriever)
{% endif %}
    llm         = build_llm()
{% if pre_retrieval.query_rewriting is not none and pre_retrieval.query_rewriting.enabled %}
    # ── Query Rewriting ──
    from {{ "langchain_core.runnables" if framework == "langchain" else "llama_index.core" }} import ...
    query = _rewrite_query(query, llm)
{% endif %}
{% if pre_retrieval.hyde is not none and pre_retrieval.hyde.enabled %}
    # ── HyDE ──
    query = _hyde_query(query, llm)
{% endif %}
    docs = retriever.invoke(query)
{% if post_retrieval.context_assembly is not none %}
    docs = _assemble_context(docs)
{% endif %}
    return _generate_answer(query, docs, llm)
```

### Canonical Ingestion Entrypoint Shape

`entrypoints/<framework>/ingestion.py.j2` handles document loading and indexing:
```python
"""{{ pipeline_name }} — Ingestion script. Generated by ragfactory."""
from __future__ import annotations
from typing import Any

{{ stages.chunking }}
{{ stages.embedding }}
{{ stages.vectordb }}


def ingest(source_path: str) -> None:
    """Load documents, chunk, embed, and store in the vector database."""
    chunker     = build_chunking()
    embedder    = build_embedding()
    vectorstore = build_vectordb(embedder)
    # Load and process documents...
```

**IMPORTANT**: `build_vectordb()` receives `embedder` as argument in both
entrypoints. Stage template `build_vectordb()` signature: `def build_vectordb(embedder: Any) -> Any:`

**IMPORTANT**: `build_retrieval()` receives `vectorstore`. `build_reranker()` receives
`retriever` and returns a wrapped retriever.

## [FIX C1] Complete Template Context Contract

### Base context (all stage templates)
```python
base_ctx = {
    "config":         RAGPipelineConfig,   # full config — use sparingly
    "framework":      str,                  # "langchain" | "llamaindex"
    "pipeline_name":  str,                  # config.name
    "dependencies":   list[str],            # get_dependencies(config) output
    "python_version": str,                  # "3.11"
}
```

### Per-stage augmented context (generator.py lines 267–322)

| Stage | Extra keys | Source |
|-------|-----------|--------|
| chunking | `chunking` | `config.indexing.chunking` |
| embedding | `embedding`, `is_late_chunking: bool` | `config.indexing.embedding`, `config.indexing.chunking.type == "late"` |
| vectordb | `vector_db` | `config.indexing.vector_db` |
| retrieval | `retrieval`, `vector_db_type: str` | `config.retrieval`, `config.indexing.vector_db.type` |
| reranker | `reranker` | `config.post_retrieval.reranker` |
| llm | `llm`, `prompt_template: str`, `advanced: AdvancedGenerationConfig \| None` | `config.generation.llm`, `config.generation.prompt_template`, `config.generation.advanced` |

### Entrypoint context (pipeline.py.j2 + ingestion.py.j2)
```python
entrypoint_ctx = {
    **base_ctx,
    "stages":         dict[str, str],       # rendered stage fragments
    "pre_retrieval":  PreRetrievalConfig,
    "post_retrieval": PostRetrievalConfig,
    "generation":     GenerationConfig,
    "ingestion":      IngestionConfig,
}
```

### Common template context
```python
common_ctx = {
    **base_ctx,
    "env_vars": list[tuple[str, str]],  # [(VAR_NAME, description), ...]
}
# docker-compose.yml additionally gets:
docker_ctx = {**base_ctx, "vector_db": VectorDBConfig}
```

**StrictUndefined rule**: Every variable used in a template MUST appear in its
context. Any extra key used (e.g., `is_late_chunking` in `jina.py.j2`) is
intentional and documented above. Templates MUST NOT access `config.*` directly
for values already extracted into a named key.

## [FIX M3] LangChain Fallback Strategies for LlamaIndex-Native Patterns

### `sentence_window` + LangChain
**Decision**: Functional approximation using `ParentDocumentRetriever`.
- `child_splitter`: `RecursiveCharacterTextSplitter(chunk_size=50)` (sentence-sized)
- `parent_splitter`: `RecursiveCharacterTextSplitter(chunk_size=window_size * 200)`
- Uses `InMemoryStore()` as docstore
- Template adds a comment:
  ```python
  # LangChain approximation of LlamaIndex SentenceWindowNodeParser.
  # child chunks (≈ sentences) are retrieved for precision;
  # parent chunks provide full context to the LLM.
  ```

### `small_to_big` + LangChain
**Decision**: Direct mapping — `ParentDocumentRetriever` is a first-class LangChain primitive.
- `child_splitter`: `RecursiveCharacterTextSplitter(chunk_size=child_chunk_size)`
- `parent_splitter`: `RecursiveCharacterTextSplitter(chunk_size=parent_chunk_size)`
- Uses `InMemoryStore()` as docstore

### `small_to_big` + LlamaIndex
**Decision**: `AutoMergingRetriever` with `HierarchicalNodeParser` —
the LlamaIndex native pattern.

## [FIX M4] Pre/Post Retrieval Wiring in pipeline.py

Pre and post retrieval logic lives exclusively in `entrypoints/<framework>/pipeline.py.j2`.
Stage templates do NOT handle this. The pipeline template has conditional blocks:

```jinja2
def run_pipeline(query: str) -> str:
    embedder    = build_embedding()
    vectorstore = build_vectordb(embedder)
    retriever   = build_retrieval(vectorstore)
{% if post_retrieval.reranker is not none %}
    retriever   = build_reranker(retriever)
{% endif %}
    llm         = build_llm()

{% if pre_retrieval.query_rewriting is not none and pre_retrieval.query_rewriting.enabled %}
    # ── Query Rewriting ({{ pre_retrieval.query_rewriting.strategy }}) ──────────
    # Generates {{ pre_retrieval.query_rewriting.num_rewrites }} rephrasings and fuses results.
    ...
    all_docs = [doc for q in queries for doc in retriever.invoke(q)]
    docs = _deduplicate(all_docs)
{% elif pre_retrieval.hyde is not none and pre_retrieval.hyde.enabled %}
    # ── HyDE — Hypothetical Document Embeddings ──────────────────────────────
    ...
    docs = retriever.invoke(hypothetical_answer)
{% else %}
    docs = retriever.invoke(query)
{% endif %}

{% if post_retrieval.context_compression is not none and post_retrieval.context_compression.enabled %}
    # ── Context Compression ({{ post_retrieval.context_compression.method }}) ─
    ...
{% endif %}

    # ── Context Assembly ──────────────────────────────────────────────────────
    context = _assemble_context(docs, {{ post_retrieval.context_assembly.max_sources }})

    return _generate_answer(query, context, llm)
```

`pre_retrieval.routing` is also handled here via a `{% if pre_retrieval.routing is not none %}` block that dispatches to named sub-pipelines.

## Invariants That Must Never Be Violated

1. **Templates never write to disk** — generator.py is pure; this plan does not change that
2. **No template accesses `config.*` directly** for values exposed as named context keys
3. **Every stage template produces exactly one `def build_<stage>()` function** — no more, no less
4. **`build_vectordb(embedder)` takes embedder** — vectorstore construction requires the embedding function
5. **`build_reranker(retriever)` takes and wraps retriever** — returns a new retriever
6. **Framework branching is ALWAYS `{% if framework == "langchain" %}...{% elif framework == "llamaindex" %}...{% endif %}`** — no other string comparisons, no `{% else %}` catch-alls
7. **All 42 templates ship inside the `ragfactory/` package** (not tests/) — `pyproject.toml` include directive already set in Phase 1b
